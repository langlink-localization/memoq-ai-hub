## What's New

- Stabilized the memoQ pre-translation flow with smaller aggregate groups, controlled provider concurrency, and clearer pending result handling.
- Added faster fallback behavior for slow aggregate jobs, including single-segment rescue and partial success responses so memoQ can fill completed segments sooner.
- Improved plugin and desktop gateway observability with request, trace, job, queue, wait, pending, timeout, and retry diagnostics.

## Notes

- This is a stability release focused on reducing long-running batch stalls and preserving successful segment results during provider slowdowns.
- Existing memoQ plugin parallelism compatibility is preserved while desktop-side scheduling controls provider throughput.
