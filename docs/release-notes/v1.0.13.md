## What's New

- Added structured-output compatibility controls for OpenAI-compatible providers so models such as DeepSeek can use `json_object` or plain text JSON prompting when `json_schema` is unavailable.
- DeepSeek-compatible provider setups now default to `json_object`, while generic OpenAI-compatible providers can automatically try `json_schema`, then `json_object`, then text.
- Builder, provider, and model settings now persist correctly when memoQ translation requests are running, preventing stale runtime snapshots from overwriting recently saved configuration.

## Validation

- Desktop tests passed locally with `pnpm --dir apps/desktop run test`.
- Release metadata was checked against `v1.0.13` before publishing.
- The release workflow continues to build the Windows portable package and stable update manifest from the tag that matches `apps/desktop/package.json`.

## Notes

- This is a compatibility and persistence patch release.
- No OpenAI SDK upgrade or DeepSeek-specific provider fork is included.
